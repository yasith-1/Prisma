Prisma ORM කියන්නේ මොකක්ද?

Prisma කියන්නේ Node.js සහ TypeScript සඳහා ඉතා ජනප්‍රිය modern database toolkit එකක්. Raw SQL ලිවීමේ සිට developer-friendly API usage දක්වා ගමන් කිරීමට Prisma සහාය වෙනවා.

💬 සරලව කිවොත් — Prisma ORM කියන්නේ ඔබ JavaScript/TypeScript code ලිවීමෙන් Database (PostgreSQL, MySQL, SQLite, MongoDB, etc.) සමඟ කතා කළ හැකි library/toolkit එකක්. SQL ඉගෙනීම නැතිව database handle කළ හැකිය.

01. Prisma vs Traditional Approaches

ක්‍රමය Code Example ගැටළු
Raw SQL SELECT * FROM users WHERE id = 1 Type safety නැහැ, Errors runtime-ලදී දැනෙයි
Traditional ORM (Sequelize) User.findOne({ where: { id: 1 } }) Complex, TypeScript support දුර්වලයි
Prisma ORM ✅ prisma.user.findUnique({ where: { id: 1 } }) Type-safe, Auto-complete, Easy to read

03. Prisma Client Setup

Application-ලදී Prisma Client use කිරීමට singleton pattern use කිරීම best practice. එය multiple connections create වීම prevent කරයි.

src/lib/prisma.ts
import { PrismaClient } from '@prisma/client'

// Global variable type declare
declare global {
  var prisma: PrismaClient | undefined
}

// Singleton pattern — development-ලදී hot reload connection issues avoid
export const prisma =
  globalThis.prisma ?? new PrismaClient({
    log: ['query', 'error', 'warn'], // SQL queries log කරනවා
  })

if (process.env.NODE_ENV !== 'production') {
  globalThis.prisma = prisma
}

💡 ඉහත code explain: globalThis.prisma ?? new PrismaClient() — already PrismaClient instance exist නම් reuse කරයි, නැත්නම් new one create කරයි. Development-ලදී hot reload ගැටළු prevent කරයි.

04. Complete Blog Schema Example

Real-world blog application-ලදී users, posts, comments, categories ඉන්නවා. ඒ සඳහා complete schema example:

prisma/schema.prisma — Blog Application
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

// User Model — blog users
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String
  password  String
  role      Role     @default(USER)
  avatar    String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  // Relations
  posts     Post[]       // User-ට posts many ඉන්නවා
  comments  Comment[]   // User-ට comments many ඉන්නවා
}

// Post Model — blog articles
model Post {
  id          Int        @id @default(autoincrement())
  title       String
  slug        String     @unique
  content     String
  excerpt     String?
  published   Boolean    @default(false)
  views       Int        @default(0)
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt

  // Foreign Key
  authorId    Int
  author      User       @relation(fields: [authorId], references: [id])

  comments    Comment[]
  categories  Category[] // Many-to-Many
}

// Comment Model
model Comment {
  id        Int      @id @default(autoincrement())
  content   String
  createdAt DateTime @default(now())

  postId    Int
  post      Post     @relation(fields: [postId], references: [id])

  authorId  Int
  author    User     @relation(fields: [authorId], references: [id])
}

// Category Model
model Category {
  id    Int    @id @default(autoincrement())
  name  String @unique
  posts Post[]
}

enum Role {
  USER
  ADMIN
  MODERATOR
}

03. Seed Data (Initial Data)

Development-ලදී test data insert කිරීමට seed script use කරයි.

prisma/seed.ts
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

async function main() {
  // Admin user create
  const admin = await prisma.user.upsert({
    where: { email: 'admin@example.com' },
    update: {},  // Already exist නම් update no
    create: {
      email: 'admin@example.com',
      name: 'Admin User',
      password: 'hashed_password',
      role: 'ADMIN',
    },
  })

  // Test posts create
  await prisma.post.createMany({
    data: [
      {
        title: 'First Post',
        slug: 'first-post',
        content: 'This is my first blog post!',
        published: true,
        authorId: admin.id,
      },
      {
        title: 'Second Post',
        slug: 'second-post',
        content: 'This is my second blog post!',
        published: false,
        authorId: admin.id,
      },
    ],
  })

  console.log('✅ Seed data successfully inserted!')
}

main()
  .catch(e => {
    console.error(e)
    process.exit(1)
  })
  .finally(async () => {
    await prisma.$disconnect()
  })

Create, Read, Update, Delete — Database සමඟ interact කිරීමේ 4 basic operations.

01. Create — Data Insert කිරීම

Create Operations
// ① Single record create
const user = await prisma.user.create({
  data: {
    email: 'kasun@example.com',
    name: 'Kasun Perera',
    password: 'hashed_pw_123',
  },
})
console.log(user.id)   // → 1
console.log(user.email) // → 'kasun@example.com'

// ② Multiple records create (bulk insert)
await prisma.user.createMany({
  data: [
    { email: 'user1@example.com', name: 'User One', password: 'pw1' },
    { email: 'user2@example.com', name: 'User Two', password: 'pw2' },
    { email: 'user3@example.com', name: 'User Three', password: 'pw3' },
  ],
  skipDuplicates: true,  // Duplicate email skip කරනවා
})

// ③ Nested create — User create + Post create (same time)
const userWithPost = await prisma.user.create({
  data: {
    email: 'writer@example.com',
    name: 'Blog Writer',
    password: 'secure_pw',
    posts: {
      create: {               // User create-ලදී post ද create
        title: 'My First Article',
        slug: 'my-first-article',
        content: 'Article content here...',
      }
    }
  },
  include: { posts: true }  // Response-ලදී posts ද include
})

// ④ Upsert — exist නම් update, නැත්නම් create
const upserted = await prisma.user.upsert({
  where: { email: 'kasun@example.com' },
  update: { name: 'Kasun Updated' },
  create: {
    email: 'kasun@example.com',
    name: 'Kasun New',
    password: 'pw'
  },
})

02. Read — Data Fetch කිරීම

Read Operations
// ① Single unique record find (by unique field)
const user = await prisma.user.findUnique({
  where: { id: 1 },          // ID by find
})
// or: where: { email: 'kasun@example.com' }

// ② First match find
const firstUser = await prisma.user.findFirst({
  where: { role: 'ADMIN' },
  orderBy: { createdAt: 'desc' },  // Latest admin
})

// ③ All records find
const allUsers = await prisma.user.findMany()

// ④ Filtered find + select specific fields
const admins = await prisma.user.findMany({
  where: {
    role: 'ADMIN',
    createdAt: {
      gte: new Date('2024-01-01'),  // 2024 Jan after
    }
  },
  select: {            // Specific fields only
    id: true,
    name: true,
    email: true,
    password: false,   // Password exclude!
  },
  orderBy: { name: 'asc' },
  take: 10,             // First 10 only
  skip: 0,              // Pagination
})

// ⑤ Include related data
const userWithPosts = await prisma.user.findUnique({
  where: { id: 1 },
  include: {
    posts: {
      where: { published: true },    // Published posts only
      orderBy: { createdAt: 'desc' },
      take: 5,
    },
    _count: { select: { posts: true } } // Total post count
  }
})

// ⑥ Count query
const totalUsers = await prisma.user.count()
const adminCount = await prisma.user.count({
  where: { role: 'ADMIN' }
})

Delete Operations
// ① Single record delete

Relations (සම්බන්ධතා)

Database tables/models ඒ අතර connections define කිරීම — One-to-One, One-to-Many, Many-to-Many.

01. One-to-Many (1:N)

Example: User-ට Posts many ඉන්නවා. (1 User → Many Posts)

User
id: Int
name: String
email: String
posts: Post[]
1 : Many
Post

authorId: Int

One-to-Many Schema
model User {
  id    Int    @id @default(autoincrement())
  name  String
  posts Post[]  // [] = "many" side
}

model Post {
  id       Int    @id @default(autoincrement())
  title    String
  authorId Int             // Foreign Key
  author   User   @relation(fields: [authorId], references: [id])
}
One-to-Many Queries
// User-ලදී Post create කිරීම
const post = await prisma.post.create({
  data: {
    title: 'My New Post',
    content: 'Content...',
    slug: 'my-new-post',
    author: {
      connect: { id: 1 }  // Existing user connect
    }
  }
})

// User-ලදී ඔක්කොම Posts include
const user = await prisma.user.findUnique({
  where: { id: 1 },
  include: {
    posts: {
      orderBy: { createdAt: 'desc' }
    }
  }
})
// user.posts → Post[] array

// Post-ලදී Author include
const postWithAuthor = await prisma.post.findUnique({
  where: { id: 1 },
  include: { author: true }
})
// postWithAuthor.author.name → 'Kasun Perera'

03. Many-to-Many (M:N)

Example: Post-ලදී Categories many ඉන්නවා, Category-ලදී Posts many ඉන්නවා. (Post ↔ Category)

Many-to-Many Schema (Implicit)
model Post {
  id         Int        @id @default(autoincrement())
  title      String
  categories Category[] // Many categories
}

model Category {
  id    Int    @id @default(autoincrement())
  name  String @unique
  posts Post[] // Many posts
}
// Prisma automatically junction table create කරනවා!
Many-to-Many Queries
// Post-ලදී categories assign
const post = await prisma.post.update({
  where: { id: 1 },
  data: {
    categories: {
      connect: [            // Multiple categories connect
        { id: 1 },
        { id: 3 },
        { name: 'Tech' }
      ]
    }
  },
  include: { categories: true }
})

// Category disconnect
await prisma.post.update({
  where: { id: 1 },
  data: {
    categories: {
      disconnect: { id: 3 }
    }
  }
})

// Category-ලදී ඇති posts find
const techPosts = await prisma.post.findMany({
  where: {
    categories: {
      some: { name: 'Tech' }  // 'Tech' category ඇති posts
    }
  },
  include: { categories: true }
})

04. Transactions

Multiple operations atomically perform කිරීම — ගෙවීම් සිදු කිරීම (Balance deduct + Payment record create) ආදී critical operations-ලදී use කරනවා.

Transactions
// ① Sequential transaction — $transaction array
const [newUser, newPost] = await prisma.$transaction([
  prisma.user.create({
    data: { email: 'new@example.com', name: 'New User', password: 'pw' }
  }),
  prisma.post.create({
    data: { title: 'Welcome Post', slug: 'welcome', content: 'Hello!', authorId: 1 }
  }),
])
// ① fail නම් ② ද rollback

// ② Interactive transaction — complex logic
await prisma.$transaction(async (tx) => {
  // User balance check
  const sender = await tx.account.findUnique({
    where: { id: 1 }
  })

  if (sender.balance < 1000) {
    throw new Error('Insufficient balance!') // → Rollback
  }

  // Deduct sender
  await tx.account.update({
    where: { id: 1 },
    data: { balance: { decrement: 1000 } }
  })

  // Add to receiver
  await tx.account.update({
    where: { id: 2 },
    data: { balance: { increment: 1000 } }
  })
})  // Error නම් automatically rollback

Error Handling

Prisma errors catch කිරීම සහ user-friendly messages return කිරීම.

01. Prisma Error Types

Error TypeCodeWhen Occurs?
PrismaClientKnownRequestErrorP2002Unique constraint violation
PrismaClientKnownRequestErrorP2025Record not found
PrismaClientKnownRequestErrorP2003Foreign key constraint violation
PrismaClientValidationError—Invalid input data
PrismaClientInitializationError—DB connection fail
Error Handling Pattern
import { Prisma } from '@prisma/client'
import { prisma } from './lib/prisma'

async function createUser(email: string, name: string) {
  try {
    const user = await prisma.user.create({
      data: { email, name, password: 'default' }
    })
    return { success: true, user }

  } catch (error) {

    if (error instanceof Prisma.PrismaClientKnownRequestError) {
      switch (error.code) {
        case 'P2002':
          // Unique constraint — Email already exists
          return {
            success: false,
            error: 'මෙම email address දැනටමත් registered. වෙනත් email use කරන්න.'
          }

        case 'P2025':
          // Record not found
          return {
            success: false,
            error: 'Record not found.'
          }

        case 'P2003':
          // Foreign key violation
          return {
            success: false,
            error: 'Related record does not exist.'
          }
      }
    }

    if (error instanceof Prisma.PrismaClientValidationError) {
      return {
        success: false,
        error: 'Invalid data provided.'
      }
    }

    // Unknown error
    console.error('Unexpected error:', error)
    return { success: false, error: 'Server error occurred.' }
  }
}

Best Practices

Professional projects-ලදී follow කළ යුතු important patterns.

✅
Singleton Pattern Use කරන්න — Development-ලදී hot reload time PrismaClient multiple instances create වෙනවා. globalThis.prisma pattern use කිරීමෙන් prevent කළ හැකිය.
✅
Select Fields Explicitly — Password, secret tokens include නොකරන්නට select use කරන්න. Default-ලදී all fields return වෙනවා — security risk!
✅
Pagination Always — findMany() ලිවීමේදී take limit add කරන්නම. නැත්නම් ලක්ෂ records එකවර load වේ.
✅
$disconnect() Call — Scripts/Lambda functions-ලදී prisma.$disconnect() finally block-ලදී call කරන්න. Connection leaks prevent කරයි.
✅
Index Add කරන්න — Frequently filtered fields-ලදී @@index([email, createdAt]) schema-ලදී add කිරීමෙන් query speed improve වෙනවා.
✅
Transactions Use — Multiple related operations (payment, inventory update, etc.) transactions-ලදී wrap කරන්න. Error නම් all rollback.
❌
N+1 Problem Avoid — Loop-ලදී per-item database query NEVER කරන්නේ. include use කිරීමෙන් single query-ලදී related data fetch කරන්න.
❌
Raw SQL Over-Use Avoid — Complex analytics queries-ලදී $queryRaw use කළ හැකිය, නමුත් normal operations-ලදී Prisma API-ම use කරන්න.
